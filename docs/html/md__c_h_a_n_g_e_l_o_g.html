[Layers 2.0.0a] - 2020-XX-XX

Added

  • Rounded corner support

Changed

  • Most buttons and controls have increased padding around their content and increased spacing between each other to improve touch experience
  • Widget attributes are now applied via paint events instead of invoking stylesheets
  • Many elements use rounded corners by default

[Layers 1.11.0a] - 2020-04-20

Added

  • Added an "Apply" button in the Customize menu that allows you to apply your customizations to the current theme of the main window.
  • Added theme saving which occurs automatically when themes are created and when customizations have been applied to them

Changed

  • Changed New theme button to be graphic only, changed its graphic to be a plus symbol, and moved it to the left of the Theme combobox
  • Moved the Customize button out of the Theme combobox and placed the button to the right of the combobox
  • Changed Delete theme button to be graphic only and moved it to the right of the Customize button
  • The New theme button works
  • The Delete theme button works
  • The New and Delete theme buttons are only enabled when a non-built-in theme is selected
  • Editing the theme name is disabled for built-in themes
  • Removed the "Save" and "Save As..." buttons from the Themes settings panel and will automatically save themes when they are created and when customizations have been applied
  • Removed everything below the Theme combobox. Changing the theme name will now be done by double clicking on the Theme combobox where it will temporarily turn into a line edit.

[Layers 1.3.0a] - 2020-03-20

Added

  • Copy operations to the main window and its child objects
  • Began work on the Customize menu
    • Created the skeleton for navigating the element hierarchy that displays the proper options on the Customize menu's sidebar. As you go down the hierarchy from the "Window", a set of labels are displayed above the preview window to inform the user specifically which element they are manipulating. These labels also act as buttons to allow the user to return to elements up the hierarchy.
    • The first customizable attribute in this release is the Titlebar's color. However, there is no structure yet to handle applying the theme changes to the main window, so there are no permanent changes.
  • Option to recursively disable primary functionality of the main window (intended for the Customize menu's copy of the window)

Changed

  • Removed the "Window Border" button in the Themes settings panel in favor of a button with a Wrench icon located within the Themes combobox used to universally access the Customize menu
  • Removed the temporary Square and Circle settings tabs/panels
  • Stability improvements

[Layers 1.2.0a] - 2020-02-23

Added

  • A "Use default" toggle switch in the themes settings panel that will force the application to use the default theme from the layers' settings file. Changing the theme in the combobox will automatically toggle this switch off.
  • Applications now watch the layers settings file for changes. This allows for the default theme to update on all apps when setting it in just one.

Changed

  • Removed a large amount of free store allocations and created destructors for objects responsible for deleting their free store objects. This should increase performance and make memory leaks far less likely.

[Layers 1.1.0a] - 2020-02-16

Added

  • Saving layers settings. The settings file is found in: "C:/Users/{Your username}/AppData/Local/Layers"
    • Fow now, the only layers setting is the default theme setting
  • Saving application settings. The settings file is found in: "C:/Users/{Your username}/AppData/Local/Layers/{Application Name}"
    • Now the theme you set for an application is reapplied when you relaunch the application.
  • New Message Box system
  • "Set as default" button within the theme combobox. Clicking this will present a confirmation message box to set the default theme. New applications will use the default theme until the user changes that application's theme manually using the theme combobox.

Fixed

  • Fixed font inconsistencies
  • The text in the theme name line edit within the themes settings panel now updates to the theme name upon switching themes. Changing the text in the line edit, however, does not yet affect the themes.

Changed

  • Overhauled the theming system
    • Most changes are structural and do not change the visual appearance of the widgets.
    • Scrollbar theming is now working
  • Changed some colors in the Royal Blue and Purple themes
  • Adjusted spacing of elements in Themes settings panel

[Layers 1.0.0a] - 2019-12-18

Added

  • This release serves as the first versioning tag for this software. It is labeled as x.x.xa because it is not feature-complete, so therefore it is considered in the alpha stage. The following points briefly cover the features currently implemented:
    • A window-scaling border
    • A theming system
    • SVG support
    • A menuing/sub-menuing system
    • A menu navigation and labeling system
    • A settings menu accessible via a titlebar icon
