\doxysubsection{Detailed Description}
The \mbox{\hyperlink{class_layers_1_1_s_v_g}{SVG}} class provides representation for \mbox{\hyperlink{class_layers_1_1_s_v_g}{SVG}} files in Layers.

An \mbox{\hyperlink{class_layers_1_1_s_v_g}{SVG}} loads an \mbox{\hyperlink{class_layers_1_1_s_v_g}{SVG}} file into a string. To make the \mbox{\hyperlink{class_layers_1_1_s_v_g}{SVG}} appear on the screen, the string is passed to QSvg\+Widget\+::load(). The load function can be called indefinitely. Therefore, changes can be made to the \mbox{\hyperlink{class_layers_1_1_s_v_g}{SVG}} string and loaded again, allowing for theme application.

Manipulating the \mbox{\hyperlink{class_layers_1_1_s_v_g}{SVG}} string itself would be punishing. To make this easier, a list of \mbox{\hyperlink{class_layers_1_1_s_v_g}{SVG}} elements is built from the string with build\+\_\+svg\+\_\+elements\+\_\+list().

The \mbox{\hyperlink{class_layers_1_1_s_v_g}{SVG}}\textquotesingle{}s constructors wait to call Themeable\+::init\+\_\+themeable() until after the \mbox{\hyperlink{class_layers_1_1_s_v_g}{SVG}} elements list has been built. This is because the \mbox{\hyperlink{class_layers_1_1_s_v_g_a51772ec69b6e9ba794914b69e9010d7e}{SVG\+::init\+\_\+attributes()}} function, which is called virtually by Themeable\+::init\+\_\+themeable(), depends on the elements list.

When a theme is applied to the \mbox{\hyperlink{class_layers_1_1_s_v_g}{SVG}}, the attributes are updated to values provided in the applied theme, but the \mbox{\hyperlink{class_layers_1_1_s_v_g}{SVG}} string needs to be updated with the new attribute values and passed along to QSvg\+Widget\+::load() before the appearance of the \mbox{\hyperlink{class_layers_1_1_s_v_g}{SVG}} updates. This class implements update\+\_\+theme\+\_\+dependencies() to ensure that this all happens after theme application.

Then, the \mbox{\hyperlink{class_layers_1_1_s_v_g}{SVG}} string is rebuilt from those elements.
