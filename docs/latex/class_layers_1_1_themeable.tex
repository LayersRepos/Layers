\doxysubsection{Detailed Description}
Provides compatibility and structure for the Layers theme system.

The \mbox{\hyperlink{class_layers_1_1_themeable}{Themeable}} class is designed to be wrapped with QWidget classes to allow them to be themed in a Layers application. A themeable builds a theme tag from details provided during initialization. The tag is used to retrieve attributes from themes.

Before a theme can be applied to a themeable, the themeable must have a name. This is because a name is the minimum requirement to construct the theme tag. To set a themeable\textquotesingle{}s name, use \mbox{\hyperlink{class_layers_1_1_themeable_a78f9b10993fcef1557aeba1bc75e5c90}{set\+\_\+name()}}.

Themes are applied to themeables recursively through \mbox{\hyperlink{class_layers_1_1_themeable_a1cf931b355c0d658d5c913af8e49edb6}{apply\+\_\+theme()}}. To do this, references to child themeables need to be stored. These references are stored through \mbox{\hyperlink{class_layers_1_1_themeable_ad89c09e860d259ab5752df92a989cd02}{store\+\_\+child\+\_\+themeable\+\_\+pointer()}}.

Before a themeable can be customized in a Layers application, two requirements must be fullfilled\+:
\begin{DoxyEnumerate}
\item A proper name must be defined with \mbox{\hyperlink{class_layers_1_1_themeable_a00802c07a93a1eefdbcf74a5c04b863b}{set\+\_\+proper\+\_\+name()}}
\item Classes that implement \mbox{\hyperlink{class_layers_1_1_themeable}{Themeable}} need to define setup\+\_\+customize\+\_\+panel() 
\end{DoxyEnumerate}

Assigns tag prefixes from the parent and the parent\textquotesingle{}s name.

This function works recursively to assign tag prefixes to the caller and children in the caller\textquotesingle{}s hierarchy. Each themeable\textquotesingle{}s name gets added to the tag prefix list as this function gets called down the hierarchy.

If no arguments are given, then assignment to the caller is skipped and the children have their prefixes assigned. Updates things that depend on the theme. Called by \mbox{\hyperlink{class_layers_1_1_themeable_a1cf931b355c0d658d5c913af8e49edb6}{apply\+\_\+theme()}}.

This function can be defined by implementers of the \mbox{\hyperlink{class_layers_1_1_themeable}{Themeable}} class to handle anything they want to change when a theme gets applied that can\textquotesingle{}t be changed through attributes. However, it is not required to implement this function.

Initializes the attributes.

This is a pure virtual function that must be defined by other classes that inherit the \mbox{\hyperlink{class_layers_1_1_themeable}{Themeable}} class. This function should be used to define a themeable\textquotesingle{}s attributes with calls to set\+\_\+attribute\+\_\+value().

This function is called by init\+\_\+themeable().

Initializes the customize panel.

{\footnotesize\ttfamily QString \& Themeable\+::tag (\begin{DoxyParamCaption}{ }\end{DoxyParamCaption})}

Get the theme tag.

The theme tag is created structurally upon calling this function. The first part of the tag is the primary prefix which determines the owning application of the themeable. It is constructed as \textquotesingle{}app/{\itshape application} {\itshape name/\textquotesingle{}}. If the themeable is built-\/in with Layers, and not application specific, then the primary prefix is constructed as \textquotesingle{}layers/\textquotesingle{}.

The tag is then followed by the tag prefixes. The prefixes are made up of parent themeable names going all the way up the hierarchy from the calling themeable. They are separated by slashes in the tag.

The last part of the tag is the themeable\textquotesingle{}s name.
