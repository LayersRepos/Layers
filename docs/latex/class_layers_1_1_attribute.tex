\doxysubsection{Detailed Description}
\mbox{\hyperlink{class_layers_1_1_attribute}{Attribute}} implementation used throughout Layers, especially by Themes and Themeables.

An \mbox{\hyperlink{class_layers_1_1_attribute}{Attribute}} represents labeled \mbox{\hyperlink{class_layers_1_1_data}{Data}}, and if the \mbox{\hyperlink{class_layers_1_1_data}{Data}} it represents is stateful, the \mbox{\hyperlink{class_layers_1_1_attribute}{Attribute}} is also responsible for defining the active state.

A key feature of Attributes is entanglement. \textquotesingle{}Entanglement\textquotesingle{} is the term used to describe the action of forcing one \mbox{\hyperlink{class_layers_1_1_attribute}{Attribute}} to delete its \mbox{\hyperlink{class_layers_1_1_data}{Data}} and point to the \mbox{\hyperlink{class_layers_1_1_data}{Data}} of another \mbox{\hyperlink{class_layers_1_1_attribute}{Attribute}}. This is done by calling \mbox{\hyperlink{class_layers_1_1_attribute_ae0a1b713db41429200ed97c399a234ba}{entangle\+\_\+with()}}.

Attributes employ a concept of ownership over the \mbox{\hyperlink{class_layers_1_1_data}{Data}} they point to. Initially, an \mbox{\hyperlink{class_layers_1_1_attribute}{Attribute}} owns its \mbox{\hyperlink{class_layers_1_1_data}{Data}}, but if it becomes {\itshape entangled} with another \mbox{\hyperlink{class_layers_1_1_attribute}{Attribute}}, it will no longer own \mbox{\hyperlink{class_layers_1_1_data}{Data}} since the \mbox{\hyperlink{class_layers_1_1_data}{Data}} it points to after being entangled is owned by the other \mbox{\hyperlink{class_layers_1_1_attribute}{Attribute}}.

{\footnotesize\ttfamily void Attribute\+::entangle\+\_\+with (\begin{DoxyParamCaption}\item[{\mbox{\hyperlink{class_layers_1_1_attribute}{Attribute}} \&}]{attribute }\end{DoxyParamCaption})}

Forces the \mbox{\hyperlink{class_layers_1_1_attribute}{Attribute}} to point to the \mbox{\hyperlink{class_layers_1_1_data}{Data}} of another \mbox{\hyperlink{class_layers_1_1_attribute}{Attribute}}.

After this function, the caller \mbox{\hyperlink{class_layers_1_1_attribute}{Attribute}} is considered {\itshape entangled} with the other \mbox{\hyperlink{class_layers_1_1_attribute}{Attribute}}.

If the caller \mbox{\hyperlink{class_layers_1_1_attribute}{Attribute}} was not previously entangled, the \mbox{\hyperlink{class_layers_1_1_data}{Data}} is deleted before the new pointer is assigned.

Establishes a new data connection and emits both entangled() and value\+\_\+changed().

Another connection is established so that if the supplied attribute becomes entangled, this function gets called again so the caller \mbox{\hyperlink{class_layers_1_1_attribute}{Attribute}} can get a pointer to the new \mbox{\hyperlink{class_layers_1_1_data}{Data}}.
