\doxysubsection{Detailed Description}
The \mbox{\hyperlink{class_layers_1_1_s_v_g___widget}{SVG\+\_\+\+Widget}} class provides representation for SVG files in Layers.

An \mbox{\hyperlink{class_layers_1_1_s_v_g___widget}{SVG\+\_\+\+Widget}} loads an SVG file into a string. To make the SVG appear on the screen, the string is passed to QSvg\+Widget\+::load(). The load function can be called indefinitely. Therefore, changes can be made to the SVG string and loaded again, allowing for theme application.

Manipulating the SVG string itself would be punishing. To make this easier, a list of SVG elements is built from the string with build\+\_\+svg\+\_\+elements\+\_\+list().

The \mbox{\hyperlink{class_layers_1_1_s_v_g___widget}{SVG\+\_\+\+Widget}}\textquotesingle{}s constructors wait to call \mbox{\hyperlink{class_layers_1_1_themeable_afdae3d404debc794625aa2f633ec23bc}{Themeable\+::init\+\_\+themeable()}} until after the SVG elements list has been built. This is because the \mbox{\hyperlink{class_layers_1_1_s_v_g___widget_ab0fd5c3a7ae76d4a4cc89b85ad9773b1}{SVG\+\_\+\+Widget\+::init\+\_\+attributes()}} function, which is called virtually by \mbox{\hyperlink{class_layers_1_1_themeable_afdae3d404debc794625aa2f633ec23bc}{Themeable\+::init\+\_\+themeable()}}, depends on the elements list.

When a theme is applied to the \mbox{\hyperlink{class_layers_1_1_s_v_g___widget}{SVG\+\_\+\+Widget}}, the attributes are updated to values provided in the applied theme, but the SVG string needs to be updated with the new attribute values and passed along to QSvg\+Widget\+::load() before the appearance of the \mbox{\hyperlink{class_layers_1_1_s_v_g___widget}{SVG\+\_\+\+Widget}} updates. This class implements \mbox{\hyperlink{class_layers_1_1_s_v_g___widget_a2b355b6d833de919c59340b75dc57941}{update\+\_\+theme\+\_\+dependencies()}} to ensure that this all happens after theme application.
